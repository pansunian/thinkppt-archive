# ThinkPPT Notion 字段结构

ThinkPPT 第一版定位为“互联网平台 IP 方案年鉴 / 方案展”。Notion 仍然是后台，线上继续通过原来的 Notion API 读取 `方案专辑` 数据库。

这版先聚焦 PDF 方案本身：封面、洞察、主题、脉络、关键页、下载。暂时不展开活动执行物料、宣传物料、执行效果和战报。

## 录入原则

1. 一个 Notion 页面仍然代表一份 PDF 方案。
2. 多份 PDF 通过同一个 `IP名称` 聚合为一个 IP。
3. 同一个 IP 不同年份用 `Year` 和 `IP阶段` 区分。
4. 前端列表看 `IP名称`，详情弹窗看 `年度主题`、`核心洞察`、`Big Idea`、`方案脉络` 和精选页。
5. PDF 页图不要在前端叠大段文字，避免和原 PPT 页面文字冲突；策展文字放在旁侧信息区。
6. `封面图OSS`、`页面图OSS目录`、`PDF原件OSS` 可以先空着，后续迁移图片时逐批补。
