# ThinkPPT Notion 字段结构

ThinkPPT 的新定位是“平台 IP 营销观察库”。Notion 仍然作为后台，前端会优先读取下面这些字段；已有字段可以继续保留。

## 必填字段

| 字段名 | 类型 | 用途 |
| --- | --- | --- |
| 标题 | Title | 单份方案或资料标题 |
| 平台 | Select | 抖音、快手、小红书、B站、微博等 |
| IP名称 | Select 或 Rich text | 例如快手春节、抖音节气、小红书营销 IP |
| 年份 | Select 或 Number | 2023、2024、2025、2026 |
| 项目类型 | Select | 招商案、执行案、复盘案、宣传物料、战报 |
| 分类 | Select | 保留原有分类筛选，例如营销方案、行业报告、品牌手册 |

## 建议字段

| 字段名 | 类型 | 用途 |
| --- | --- | --- |
| 档案类型 | Select | 招商 PDF、KV、海报、视频截图、活动页、战报 |
| 持续年限 | Select | 第1年、第2年、第3年、长期 IP |
| 行业 | Select | 快消、美妆、酒水、文旅、汽车、电商、本地生活 |
| 品牌 | Rich text 或 Select | 平台、客户或项目关联品牌 |
| 合作品牌 | Multi-select 或 Rich text | 已公开合作品牌 |
| 招商权益 | Multi-select 或 Rich text | 资源位、达人、直播、内容共创、站内曝光等 |
| 主题Slogan | Rich text | 当年主标题或活动主题 |
| 关键变化 | Rich text | 相比上一年的主题、权益、视觉或招商逻辑变化 |
| 执行效果 | Rich text | 公开曝光、互动、话题、战报摘要 |
| 站长观察 | Rich text | 你的判断：为什么值得看，适合谁参考 |
| 标签 | Multi-select | 节点营销、平台招商、春节、种草、晚会等 |
| 方案页数 | Number 或 Rich text | 用于卡片显示 |
| 文件大小 | Number 或 Rich text | 用于卡片显示 |
| 下载链接 | URL | PDF 或资料下载链接 |
| 编辑推荐 | Checkbox | 首页优先展示 |

## 录入示例

| 字段 | 示例 |
| --- | --- |
| 标题 | 快手2026CNY招商项目通案 |
| 平台 | 快手 |
| IP名称 | 快手春节 IP |
| 年份 | 2026 |
| 项目类型 | 招商案 |
| 档案类型 | 招商 PDF |
| 持续年限 | 第4年 |
| 行业 | 节点营销 |
| 主题Slogan | 快手马年星晚 |
| 招商权益 | 晚会资源、站内曝光、达人共创、直播互动 |
| 关键变化 | 从单一曝光资源，升级为晚会内容、达人传播和电商转化的组合权益 |
| 站长观察 | 这类平台春节 IP 的核心价值不只是流量，而是把品牌放进春节内容场景里，让品牌有理由参与节点传播。 |

## 前端兼容字段名

前端已经兼容中英文别名。比如“平台”也可以写成 `Platform`，“IP名称”也可以写成 `IP Name`，“站长观察”也可以写成 `Editor's Note`。

为了后续维护简单，建议统一使用中文字段名：

`平台`、`IP名称`、`年份`、`项目类型`、`档案类型`、`持续年限`、`主题Slogan`、`关键变化`、`招商权益`、`合作品牌`、`执行效果`、`站长观察`。
